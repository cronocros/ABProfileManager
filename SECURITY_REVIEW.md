# ABProfileManager Security Review

검토 날짜:
- 2026-03-07

검토 범위:
- 문자열 가져오기 경로
- 퀘스트 대량 포기 경로
- 템플릿 이름 입력 경로
- 상태/명령/외부 입력 처리

## 결론

현재 기준으로 즉시 악용될 만한 고위험 코드 실행 경로는 확인되지 않았습니다.

확인된 사실:
- `loadstring`, `RunScript` 같은 동적 코드 실행 없음
- 외부 네트워크 전송 기능 없음
- 템플릿 문자열 가져오기는 실행 코드가 아니라 데이터 파싱 방식

## 이번에 반영한 보강

### 1. 문자열 가져오기 입력 검증 강화

- 최대 입력 길이 제한
- 최대 줄 수 제한
- 중복 슬롯 행 차단
- 허용된 액션 종류만 수용
- 템플릿 이름 길이 제한
- 제어 문자 제거 및 단일행 입력 정화

목적:
- 비정상적으로 큰 입력으로 SavedVariables나 UI를 오염시키는 문제 방지
- 잘못된 import 문자열로 이상 상태가 생기는 문제 방지

### 2. 템플릿 이름 정화

- 저장
- 복제
- 삭제
- 조회
- import override

모든 경로에서 단일행 기준으로 정화 처리

목적:
- 개행/제어문자/이상 공백이 UI나 저장 구조를 깨뜨리는 문제 방지

### 3. 전체 퀘스트 포기 강제 확인

- 설정에서 일반 확인창을 꺼도
- `전체 퀘스트 포기`는 항상 확인 모달을 거침

목적:
- 파괴적 작업의 실수 실행 방지

## 현재 남은 저위험 메모

1. 정적 문법 검사
- 이 환경에는 `lua/luac`가 없어 자동 검사를 못 돌림

2. WoW 클라이언트 API 편차
- 퀘스트 선택/포기 API는 클라이언트 상태와 패치에 따라 세부 동작이 다를 수 있음
- 현재는 `questID` 우선, 로그 인덱스 fallback으로 보강됨

3. 문자열 import는 신뢰 가능한 사용자 데이터 기준
- 실행 코드는 아니지만, 여전히 임의 문자열 입력 경로이므로 현재처럼 길이/형식 검증을 유지해야 함

## 유지 원칙

- import는 코드 실행이 아니라 데이터 파싱만 허용
- 파괴적 작업은 항상 확인 모달 우선
- UI에 표시되는 이름/텍스트는 단일행 정화 우선
- 앞으로도 신규 외부 입력 경로가 생기면 길이 제한과 타입 검증을 먼저 넣을 것
