# ABProfileManager Security Review

검토 날짜:
- 2026-03-12

검토 범위:
- 문자열 가져오기 경로
- 퀘스트 대량 포기 경로
- 템플릿 이름 입력 경로
- profession 자동 추적 데이터 경로
- 지도 오버레이와 설정 패널 추가 UI 경로
- 상태/명령/외부 입력 처리

## 결론

현재 기준으로 즉시 악용될 만한 고위험 코드 실행 경로는 확인되지 않았다.

확인된 사실:
- `loadstring`, `RunScript` 같은 동적 코드 실행 없음
- 외부 네트워크 전송 기능 없음
- 템플릿 문자열 가져오기는 실행 코드가 아니라 데이터 파싱 방식
- profession 자동 추적은 내장 데이터셋 + 완료 퀘스트 조회 기반
- Midnight 지도 오버레이는 정적 좌표 데이터 기반

## 주요 보강 사항

### 1. 문자열 가져오기 입력 검증 강화

- 최대 입력 길이 제한
- 최대 줄 수 제한
- 중복 슬롯 행 차단
- 허용된 액션 종류만 수용
- 템플릿 이름 길이 제한
- 제어 문자 제거 및 단일행 입력 정화

목적:
- 비정상적으로 큰 입력으로 SavedVariables나 UI를 오염시키는 문제 방지
- 잘못된 import 문자열로 이상 상태가 생기는 문제 방지

### 2. 템플릿 이름 정화

- 저장
- 복제
- 삭제
- 조회
- import override

모든 경로에서 단일행 기준으로 정화 처리

목적:
- 개행/제어문자/이상 공백이 UI나 저장 구조를 깨뜨리는 문제 방지

### 3. 전체 퀘스트 포기 강제 확인

- 설정에서 일반 확인창을 꺼도
- `전체 퀘스트 포기`는 항상 확인 모달을 거침

목적:
- 파괴적 작업의 실수 실행 방지

### 4. profession 자동 추적 입력 경계

- profession 소스 정의는 내장 Lua 테이블만 사용
- 완료 여부는 퀘스트 완료 플래그 / 숨은 퀘스트 조회 결과만 소비
- 외부 문자열이나 임의 코드 경로를 실행하지 않음
- 한국어 퀘스트명 표시도 클라이언트 API 반환값 또는 내장 번역 규칙만 사용

목적:
- profession 자동 추적 추가 기능이 새 실행 경로나 임의 입력 경로를 만들지 않게 유지

### 5. 설정 하위 카테고리와 지도 오버레이

- 와우 `설정 > 애드온` 하위 카테고리는 경량 패널만 등록
- 기존 메인 UI 프레임을 재부모화하지 않음
- 지도 오버레이는 내장 좌표와 라벨만 사용하고 외부 입력을 받지 않음

목적:
- 레이아웃 오류와 예기치 않은 참조 공유를 줄이고, UI 안전성을 유지

## 현재 남은 저위험 메모

1. 정적 문법 검사
- 이 환경에는 `lua`/`luac`가 없어 Blizzard 런타임과 동일한 컴파일 검사는 못 한다.
- 대신 `luaparser` 기반 전체 Lua 파싱을 실행해 문법 오류를 점검한다.

2. WoW 클라이언트 API 편차
- 퀘스트 선택/포기 API는 클라이언트 상태와 패치에 따라 세부 동작이 다를 수 있다.
- profession 퀘스트명 API도 일부 숨은 퀘스트에서는 제목을 주지 않을 수 있다.

3. 지도 좌표 신뢰도
- Midnight 지도 오버레이는 정적 좌표 기반이므로, 패치 후 좌표 드리프트가 생기면 수동 보정이 필요하다.

## 유지 원칙

- import는 코드 실행이 아니라 데이터 파싱만 허용
- 파괴적 작업은 항상 확인 모달 우선
- UI에 표시되는 이름/텍스트는 단일행 정화 우선
- 신규 외부 입력 경로가 생기면 길이 제한과 타입 검증을 먼저 넣을 것
- profession 및 지도 데이터는 내장 데이터셋 기반을 유지할 것
